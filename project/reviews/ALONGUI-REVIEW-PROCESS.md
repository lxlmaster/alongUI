# ▎alongUI · Review 流程与记录

> 你的角色：独立 Reviewer — 只审查，不改代码，记录一切

---

## 你的工作流

每次 Codex 完成开发后，按以下流程走：

### Step 1 · 视觉审查

打开 Codex 的产出（原型页面 / 组件），对照 `../design/ALONGUI-APPLE-SPEC.md` 逐条检查：

```
□ 颜色是否全部来自 Apple 色板？
□ 有没有出现蓝色主色、白色输入框、4px 圆角等典型错误？
□ 圆角是否匹配组件大小？（Button 6px / Card 8px / Modal 12px）
□ 阴影是否克制？（透明度 ≤ 0.1）
□ 字体栈是否正确？
□ Focus 样式是不是黑色而不是蓝色？
□ 暗色模式是否实现了？
```

### Step 2 · 功能审查

```
□ 组件 Props 是否与 `../planning/ALONGUI-COMPONENTS.md` 一致？
□ v-model 双向绑定是否正常工作？
□ 禁用态 / loading 态是否正确？
□ Slots 是否按设计文档实现？
□ 键盘交互（Tab / Enter / Escape）是否可用？
```

### Step 3 · 记录 Review 文档

每次 Review 输出一份 Review Report，存到 `D:\codex\alongUI\project\reviews\reports\` 目录下。
