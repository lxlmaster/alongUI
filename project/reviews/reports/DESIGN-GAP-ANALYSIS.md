# alongUI · 与 iOS/macOS 的设计差距分析

> 当前状态：给 Element Plus 换了件 Apple 颜色的外衣
> 目标状态：组件看起来像 Apple 做的

---

## 一、根本问题

```
当前思路：设计一个按钮 → 选颜色 → 用 #1D1D1F 代替 #409EFF
Apple 思路：观察 iOS 按钮长什么样 → 按它的结构做

我们只改了颜色，没改结构。
```

---

## 二、逐个组件的差距

### 1. Button 按钮

```
当前 alongUI                          iOS 原生
┌──────────────────┐                 [主要操作]
│  主要按钮         │                 [次要操作]
└──────────────────┘                 

我们的：
• 36px 高度，圆角 6px
• 填充背景 #1D1D1F
• hover 变灰色
• 有 min-width、padding
• 在 Element Plus 的骨架上改颜色

iOS 的：
• 纯文字，无背景、无边框
• tint color (#007AFF) 区分可交互
• 没有 hover（触屏无 hover）
• 重要操作用粗体文字 (Semibold)
• 删除操作用红色文字
• 无 min-width，文字撑开

macOS 的：
• 纯文字按钮（类似 iOS）
• 部分场景灰色胶囊按钮（NSPopover 中的操作）
```

**差距：我们的按钮像"Web 管理系统按钮涂黑"，不是 "Apple 按钮"**

### 2. Input 输入框

```
当前 alongUI                          iOS 原生
┌──────────────────┐            ┌──────────────────┐
│                   │            │                   │
│  姓名             │            │   请输入姓名       │
│                   │            │                   │
└──────────────────┘            └──────────────────┘

我们的：
• 灰色背景 #F5F5F7
• 4px 边框 #D2D2D7
• 聚焦边框变黑
• 36px 高度

iOS 的：
• 白色背景
• 底部横线或圆角矩形
• 聚焦时蓝色 tint 边框
• 48px 高度（触摸目标） 
• 圆角 10pt（iOS 原生输入框是圆角矩形样式）
```

**差距：输入框最明显的——Apple 输入框是白色底的圆角矩形，不是灰色底的 Web 输入框**

### 3. Switch 开关

```
当前 alongUI                          iOS 原生
┌──────┐                            ┌──────────┐
│ ●    │                            │    ●     │
└──────┘                            └──────────┘

我们的：
• 44x24px，灰色/黑色切换
• 滑块白色
• 开启态黑色

iOS 的：
• 51x31pt（标准 iOS 开关）
• 开启态绿色 (#34C759)
• 滑块带轻微投影
• 关闭态白色边框底
```

**差距：iOS Switch 是标志性的绿色，我们的黑色 Switch 看起来像 Android 风格**

### 4. 列表 / Table

```
我们的 Table：
• 表头 12px 大写
• 行分割线 #F2F2F7
• hover 变灰
• 单元格 13px

iOS TableView：
• 无表头（或者大号 section header）
• 单元格高度 44pt
• 右侧小箭头 >
• 点击整行高亮

macOS NSTableView：
• 交替行色
• 选中行蓝色高亮
• 无 hover 效果
```

### 5. 导航 / Menu

```
我们的 Menu：
• 侧栏 240px，竖排列表
• 选中项黑色
• #EBEBF0 背景
• hover 变灰

iOS TabBar：
• 底部，5个tab
• 图标+文字
• 选中 tint color

macOS Sidebar：
• 选中项蓝色高亮 (#007AFF)
• 3pt 左侧高亮条
• 毛玻璃背景
```

### 6. Dialog / 弹窗

```
我们的 Dialog：
• 白色卡片，12px 圆角
• 标题+内容+底部按钮
• 遮罩 rgba(0,0,0,0.3)

macOS Alert：
• 毛玻璃面板
• 图标（警告/错误/信息）
• 按钮并排
• 无遮罩
```

## 三、最核心的差异

| 维度 | 我们现在 | iOS/macOS 应该是 |
|------|---------|-----------------|
| **按钮风格** | 填充矩形（Element 风格） | **文字按钮为主**，填充仅用于极端强调 |
| **主色用途** | 填充按钮背景 | **tint color**，作用于文字/图标/边框 |
| **输入框** | 灰色底+边框 | **白色底**，圆角矩形，iOS 用 tint 边框 |
| **圆角** | 6px 统一下 | **iOS 10pt，macOS 4-6pt**，视觉一致性 |
| **交互反馈** | hover 变色 | **iOS 无 hover，有点击高亮** |
| **阴影** | rgba(0,0,0,0.04-0.1) | **极淡或不用** |
| **视觉层次** | 靠边框+阴影区分 | **靠毛玻璃+透明度分层** |

---

## 四、需要改的方向

### 急迫（不改不像 Apple）

1. **Button 完全重做** — 去掉填充按钮作为默认样式，改为文字 tint color 风格
2. **Input 改白底** — 白色背景 10pt 圆角，聚焦用 tint color 光晕
3. **Switch 改绿色** — 开启态 #34C759（iOS 标准绿）
4. **引入 tint color** — #007AFF 作为交互色（按钮/链接/选中/焦点），黑色仅做文字色

### 重要（区分 Web 感）

5. **Focus 从 "黑色 outline" 改成 "tint color glow"** — 发光的 focus ring 更像 Apple
6. **macOS 侧栏** — 选中用 3pt 左侧高亮条
7. **毛玻璃应用** — 导航栏、侧栏、浮动面板用 backdrop-filter

### 优化（润色）

8. **间距放大** — iOS 的留白比 Web 慷慨
9. **字号微调** — iOS 按钮用 17pt，我们的 14pt 太小
10. **动效曲线** — 加入弹性曲线 spring()
